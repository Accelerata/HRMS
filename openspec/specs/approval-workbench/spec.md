# approval-workbench

## Purpose

为审批人提供统一的审批工作台：完整字段的待办列表、按业务类型聚合的统一审批详情，以及基于 RBAC 权限码的访问控制，支撑 HR、部门主管、财务专员在同一入口处理各类审批。

## Requirements

### Requirement: 待办列表完整字段

待办列表 SHALL 返回每条待办的：申请人姓名、申请类型（含标签）、申请时间、截止时间（dueTime）、当前步骤序号与步骤名称、业务摘要。申请人为空时 MUST 回退业务摘要中的名称。

#### Scenario: 待办字段完整返回

- **WHEN** 审批人查询待办列表
- **THEN** 每条待办包含申请人、申请类型、申请时间、截止时间、步骤名称，可直接渲染 8.2.1 列表

#### Scenario: 截止时间来自引擎计算

- **WHEN** 审批任务生成时已计算 48h 截止时间
- **THEN** 待办列表 dueTime 字段返回该截止时间

### Requirement: 统一审批详情

系统 SHALL 提供统一详情接口，按业务类型返回：业务单据完整字段（各类型的申请信息）、审批历史（各节点的审批人、动作、意见、操作时间、截止时间、是否待审）、当前用户在该单据下的可执行操作（可审批的记录 ID / 可转交）。

#### Scenario: 请假详情展示

- **WHEN** 审批人查看某请假申请详情
- **THEN** 返回请假类型、起止日期、天数、事由、交接人等申请信息，及已审批节点与意见、当前待审节点

#### Scenario: 审批历史含代审留痕

- **WHEN** 某节点为「B 代 A 审批」
- **THEN** 审批历史中该节点意见显示「B 代 A 审批：…」且记录实际操作人

#### Scenario: 当前用户可操作标识

- **WHEN** 当前用户在该单据下有待处理审批记录
- **THEN** 详情返回该记录 ID 与可执行操作（通过/拒绝/转交）；无待处理记录时返回不可操作

### Requirement: 工作台访问权限

工作台接口 SHALL 使用 `approval:workbench`/`approval:view` 权限码（RBAC 种子已有），持有任一审批类角色（HR、部门主管、财务专员）的用户 MUST 能访问；ROLE_FINANCE 的权限种子 MUST 补齐 approval 权限树以支撑薪资批次审批。

#### Scenario: 财务专员访问工作台

- **WHEN** 财务专员查询待办列表
- **THEN** 正常返回其薪资批次审批待办，不再 403

#### Scenario: 普通员工无工作台权限

- **WHEN** 无审批权限的普通员工访问工作台接口
- **THEN** 系统返回 403
